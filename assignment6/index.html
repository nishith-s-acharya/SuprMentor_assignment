01

What is Component Hunt?

Identifying and naming every reusable UI piece that makes up a website.

🧩

Component

A self-contained, reusable piece of UI — like a button, card, or navigation bar — that has its own structure and style.

📐

Structure

How components are nested and arranged on the page — the hierarchy from the outermost layout to the smallest detail.

🔁

Reusability

The same component (e.g., a movie card) appears hundreds of times on Netflix — defined once, reused everywhere.

⚙️

Why It Matters

Understanding components is the foundation of modern front-end development — React, Vue, and Angular are all component-based.

05

Key Learnings

Insights gained by hunting components on Netflix.

01

Everything is a Component

Even the smallest UI piece — a like button, a rank badge, a progress bar — is a standalone component with its own logic and style.

02

Reuse Over Repeat

The same <MovieCard /> is reused across dozens of rows. Different data (props), same structure — this is the power of components.

03

Parent Passes, Child Displays

Parent components like <ContentRow /> pass data down to <MovieCard /> children via props.

04

Components Have State

Some components manage their own data — <SearchBar /> tracks input text, <AddToListBtn /> tracks added/not-added state.

05

Layout vs. UI vs. Content

Components can be classified: Layout (structure), UI Elements (buttons, badges), Content (cards, banners), and Interactive (forms, toggles).

06

Think in Trees

Web UIs are trees — a root <App>, with branches for pages, sections, rows, and leaves for individual elements.
